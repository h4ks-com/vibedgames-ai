Glider Lab 5 — a working CA memory register

This is real digital memory built from cellular automata. Each cell is a flip-flop: two CA “territories” (orange = bit 1, blue = bit 0) that mutually annihilate where they meet, so one wins and holds. Eight cells chained = an 8-bit register. Click a cell to flip its bit, type a byte to write, and watch the CA store it indefinitely — no refresh needed. (These rules were evolved on a cluster to be perfectly stable.)

The honest story. We first tried analog memory (a recurrent “reservoir”) — it failed (the loop just oscillated, storing nothing). This digital route works: a flip-flop is a bistable latch, and a cluster search found CA rules whose latch is perfectly dominant; a thorough test confirmed a chained register holds at 100% (word-perfect) up to 32 bits and 600 steps with no decay, no refresh, and no cross-talk. The arc: reservoir ✗ → flip-flop ✓ → register ✓ → shift register ✓. Honest limit: a full CPU also needs composable logic gates, and a rigorous search found that robust, generalising gate composition does NOT emerge in this substrate — storage works, but autonomous logic is the open bottleneck.
