Why YAA?

🧠

Multi-Model Routing

Automatically routes tasks to the best model for the job. GPT-4o for reasoning, Claude for writing, local models for speed.

🔌

Plugin Architecture

Extend YAA with plugins written in any language. Drop them in and they just work. No complex build steps required.

🛡️

Sandboxed Execution

Every tool call runs in an isolated environment. Your system stays safe even when the agent goes off-script.

💾

Persistent Memory

Long-term memory across sessions. YAA remembers context, preferences, and learns from past interactions.

🔄

Self-Correction Loop

Built-in reflection cycle. When YAA makes a mistake, it catches it, backtracks, and tries again automatically.

⚙️

Zero Config Start

`npx yaa init` and you're running. No API keys wizardry, no YAML hell, no 47-step setup guide.
