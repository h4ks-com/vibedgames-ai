Wang tiles: Each tile is a colored square whose edges must match adjacent tiles' edges. With 4 colors per edge and 4 edges per tile, the theoretical max tile types is 256. We assign tile types based on the bitmap values — the tile sequence IS the data.

Key insight: The "seed" the LLM must output is the Wang tile sequence + CA initial state. That sequence contains exactly as much information as the original bitmap — no compression has occurred. The Wang tiles don't help because assigning a unique tile type per cell value is just a 1:1 encoding, not compression.

Conclusion: Even with Wang tiles + CA + GA, the "seed" the LLM outputs must contain enough bits to uniquely specify the target data. 4-color Wang tiles are Turing complete — meaning they can encode anything, but the encoding is never smaller than the data itself. This is the pigeonhole principle in action. The pipeline looks cool but achieves zero compression for high-entropy data like natural language.
